# 检索规则 (Retrieval Rules)

## 双引擎设计

| 引擎 | 适用 | 核心内容 |
|------|------|---------|
| **LanceDB** | 语义检索 | 用户偏好/项目经验/领域知识/历史决策 |
| **OpenViking** | 知识检索 | 代码知识/技术文档/API参考/最佳实践 |

## 检索意图识别

| 意图 | 特征 | 策略 |
|------|------|------|
| **记忆检索** | "上次""之前""记得" | 本地优先 |
| **网络检索** | "最新""今年""现在" | 网络优先 |

## 置信度标注

| 标注 | 分值 | 处置 |
|------|------|------|
| 🟢 高 | ≥0.8 | 直接应用 |
| 🟡 中 | 0.5–0.8 | 参考+提示不确定性 |
| 🔴 低 | <0.5 | 提示可能不准确，需确认 |

## 综合评分公式
```
final = 语义(0.4) + 关键词(0.2) + 时效(0.2) + 重要性(0.15) + 关联(0.05)
```

## 多路召回通道
- 语义通道（向量相似度）
- 关键词通道（精确匹配）
- 时序通道（时间衰减）
- 重要性通道（评分过滤）
- 关联通道（图谱关系）

## 快捷指令
| 指令 | 功能 |
|------|------|
| `/omni_recall [查询]` | 通用检索 |
| `/omni_recall code [查询]` | 代码知识 |
| `/omni_recall project [名]` | 项目记忆 |
| `/omni_recall domain [域]` | 领域知识 |
| `/omni_recall recent [N天]` | 近期记忆 |
| `/omni_recall important` | 重要记忆 |
