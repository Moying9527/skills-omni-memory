# 上下文压缩规则 (Context Compression Rules)

## 核心原则

**压缩目标**: 在保留关键信息的前提下，将上下文Token占用减少70-80%。

**压缩触发条件**:
1. 上下文Token数超过预算的70%
2. 用户显式请求压缩
3. 开始新任务前
4. 会话即将结束时

---

## 压缩策略

### 策略1: 锚定迭代摘要 (Anchored Iterative Summarization)

**适用场景**: 长对话历史、多轮任务

**方法**:
1. 识别待淘汰的消息段（而非全部历史）
2. 单独摘要该片段，保留关键锚点:
   - 决策点
   - 变更记录
   - 错误与修复
   - 下一步计划
3. 将新摘要合并到持久化的"锚点状态"

**锚点字段**:
```markdown
## Anchored State

### Decisions
- [时间]: [决策内容] → [结果]

### Changes
- [时间]: [变更内容] → [影响]

### Errors & Fixes
- [时间]: [错误描述] → [修复方法]

### Next Steps
- [优先级]: [待办事项]
```

**示例**:
```
原文（200 tokens）:
用户: 帮我写一个Python脚本
Agent: 好的，您需要什么功能？
用户: 处理CSV文件，读取数据并统计行数
Agent: 明白了，我使用pandas来实现
...（多轮对话）...

压缩后（50 tokens）:
决策: 使用pandas处理CSV
功能: 读取数据 + 统计行数
状态: 已实现，待优化
```

### 策略2: 关键信息提取 (Key Information Extraction)

**适用场景**: 技术讨论、决策记录

**提取维度**:
1. **用户偏好**: 工具选择、风格偏好、格式要求
2. **技术决策**: 架构选择、库/框架、设计模式
3. **约束条件**: 时间限制、性能要求、兼容性
4. **进度状态**: 已完成、进行中、阻塞点

**提取格式**:
```markdown
## Extracted Key Info

### Preferences
- 工具: [偏好工具]
- 风格: [代码/写作风格]
- 格式: [输出格式]

### Decisions
- [决策项]: [选择] (原因: [原因])

### Constraints
- 时间: [截止日期]
- 性能: [性能要求]
- 兼容: [兼容性要求]

### Progress
- ✅ [已完成项]
- 🔄 [进行中项]
- ⛔ [阻塞项]
```

### 策略3: 结构化压缩 (Structured Compression)

**适用场景**: 代码、配置、数据

**压缩规则**:

| 内容类型 | 压缩比例 | 保留内容 |
|----------|----------|----------|
| 历史对话（旧轮次） | 5:1 | 决策与结果 |
| 工具输出/日志 | 20:1 | 仅保留结论 |
| 代码片段 | 3:1 | 关键函数 + 注释 |
| 配置文件 | 2:1 | 关键配置项 |
| 最近消息（最后5轮） | 不压缩 | 完整保留 |
| 系统提示 | 不压缩 | 完整保留 |

## 压缩工作流程

### Step 1: 评估上下文大小

```python
# 伪代码
token_count = count_tokens(context)
token_budget = get_budget()  # 通常200K for Claude
threshold = token_budget * 0.7  # 70%触发压缩

if token_count > threshold:
    trigger_compression()
```

### Step 2: 选择压缩策略

| 场景 | 策略 | 优先级 |
|------|------|--------|
| 长对话历史 | 锚定迭代摘要 | P0 |
| 技术讨论 | 关键信息提取 | P0 |
| 代码/配置 | 结构化压缩 | P1 |
| 混合内容 | 组合策略 | P1 |

### Step 3: 执行压缩

**压缩检查清单**:
- [ ] 保留所有关键决策
- [ ] 保留用户明确偏好
- [ ] 保留当前任务状态
- [ ] 保留阻塞点和依赖
- [ ] 移除冗余对话
- [ ] 移除已解决的错误详情
- [ ] 合并相似信息

### Step 4: 验证压缩质量

**验证标准**:
1. **完整性**: 关键信息无遗漏
2. **准确性**: 无信息扭曲或误解
3. **可用性**: 压缩后内容可直接使用

**验证方法**:
- 对比压缩前后关键信息
- 检查决策链完整性
- 确认用户偏好保留

### Step 5: 更新SESSION-STATE.md

```markdown
# SESSION-STATE.md — Compressed Context

## Last Compressed
Time: YYYY-MM-DD HH:mm
Method: [压缩策略]
Ratio: [压缩比例]

## Compressed Summary
[压缩后的上下文摘要]

## Key Anchors Preserved
- 决策: [关键决策]
- 偏好: [用户偏好]
- 状态: [当前状态]
```

## 压缩质量检查

### 漂移检测

**信号**:
- 重复已完成操作
- 目标表述变化
- 上下文不一致

**处理**:
1. 检测到漂移 → 停止压缩
2. 回滚到上一稳定状态
3. 重新压缩，加强锚点保留

### 完整性验证

**必须保留的信息**:
1. 用户明确指令
2. 已确认的技术决策
3. 用户表达的偏好
4. 当前任务进度
5. 未解决的阻塞点

**可移除的信息**:
1. 已解决的中间问题
2. 探索性的讨论
3. 重复确认的内容
4. 临时性状态

## 快捷指令

| 指令 | 功能 |
|------|------|
| `/omni_compress` | 自动压缩当前上下文 |
| `/omni_compress full` | 强制全量压缩 |
| `/omni_compress anchor` | 使用锚定摘要压缩 |
| `/omni_compress extract` | 提取关键信息 |

---

## 错误处理

| 错误 | 处理 |
|------|------|
| 压缩后信息丢失 | 回滚 + 重新压缩 + 加强保留 |
| 压缩比例不足 | 切换策略 + 二次压缩 |
| 压缩后不可用 | 保留原始 + 标记待优化 |
